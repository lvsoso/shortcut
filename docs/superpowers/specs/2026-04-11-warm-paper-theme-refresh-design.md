# Warm Paper Theme Refresh Design

## 背景

当前工具站的默认视觉语言存在三个核心问题：

1. 默认主题主色偏紫，和“开发工具工作台”的产品气质不匹配，容易显得廉价。
2. 全局背景、面板和按钮大量依赖玻璃感、彩色光晕和偏冷渐变，层级不稳，视觉噪声偏高。
3. 侧栏、壳层、卡片、输入框虽然共用变量，但没有形成统一的材质感，整体更像样式拼装而不是完整主题。

本次目标不是重做页面结构，而是在保留现有组件结构和交互的前提下，重建默认主题的视觉令牌与公共样式语言，把整体体验调整为“米白书页 + 书房工作台”。

## 目标

- 将默认主题调整为温暖纸感，避免当前偏紫、偏冷、偏塑料的观感。
- 保持工具站的专业感和长期使用舒适度，不走花哨装饰路线。
- 只改全局主题令牌和公共组件样式，不改各工具模块的业务结构。
- 让浅色和深色模式都保持统一气质，而不是简单反相。

## 非目标

- 不新增第三套主题名称。
- 不改 Zustand 主题状态结构，仍保留 `default | anime` 与 `light | dark | system`。
- 不重写各工具模块内部布局，不做模块级 UI 大翻修。
- 不处理这次主题改动以外的性能、交互或功能问题。

## 设计方向

### 视觉基调

默认主题改为“米白书页 + 书房工作台”：

- 浅色模式以米白、暖灰、胡桃木棕为主，不再使用当前靛紫作为全局强调色。
- 深色模式改为暖黑褐和纸墨对比关系，避免冷蓝黑底。
- 强调色从“高饱和发亮按钮”改为“克制的棕金色操作点”，只在关键操作和焦点态中使用。

### 材质语言

- 页面背景应像纸面和桌面之间的过渡，保留轻微渐变，但去掉明显彩色光晕。
- 面板更像纸页或卡片夹层，有清晰边界和轻阴影，不再依赖强玻璃模糊。
- 侧栏保持独立块面，但变为暖色深底，与主内容形成“桌面 + 纸页”的关系。

### 排版感

- 维持当前整体结构和信息布局。
- 强化文字可读性，降低次级文字的灰紫感，回到暖灰体系。
- 保持标题、说明、按钮和输入框的一致节奏，避免每块都像不同系统。

## 方案对比

### 方案 A：只替换颜色

只改 CSS 变量中的颜色值，不动公共组件样式规则。

优点：

- 改动最小。
- 风险最低。

缺点：

- 无法解决玻璃感、阴影和边框语言不统一的问题。
- 最终效果只会“没那么丑”，但不会形成完整主题。

### 方案 B：重做默认主题令牌和公共样式

调整 `src/index.css` 中默认主题的颜色、渐变、阴影、边框，并联动公共壳层、侧栏、按钮、输入框等共用样式。

优点：

- 改动集中。
- 整体气质能统一起来。
- 不影响工具模块结构，维护成本可控。

缺点：

- 需要同步检查浅色/深色两套模式的对比关系。

### 方案 C：主题重做并顺手重整版式

在方案 B 基础上，继续大幅修改标题区、侧栏头图、按钮尺寸与局部布局比例。

优点：

- 完成度最高。

缺点：

- 范围明显扩大。
- 容易把本次需求从主题重做带偏到整站改版。

推荐方案：B。

原因：当前问题主要来自视觉令牌和公共组件样式，不是结构问题。方案 B 能直接打到根因，且不会把范围扩散。

## 实施范围

### 1. 全局主题令牌

更新 `src/index.css` 中默认主题的以下变量：

- `--color-bg-app`
- `--color-bg-panel`
- `--color-bg-card`
- `--color-bg-sidebar`
- `--color-bg-input`
- `--color-text-primary`
- `--color-text-secondary`
- `--color-text-muted`
- `--color-border-default`
- `--color-border-strong`
- `--color-accent`
- `--color-accent-hover`
- `--color-accent-soft`
- `--shadow-card`
- `--shadow-focus`
- `--gradient-app`
- `--gradient-accent`

其中 `anime` 主题不作为本次主改对象，只确保不被误伤。

### 2. 公共壳层样式

重设这些类的视觉规则：

- `.app-shell-panel`
- `.app-shell-panel::before`
- `.app-shell-subtle-panel`
- `.app-shell-hairline`

目标是：

- 降低磨砂玻璃感和发光感。
- 减少高饱和彩色叠层。
- 让主壳层更接近纸页面板。

### 3. 侧栏视觉

针对 [Sidebar.tsx](/Users/Zhuanz/lvsoso/tools/src/components/layout/Sidebar.tsx) 的现有结构，只调整样式表达：

- 侧栏深底从冷蓝黑改为暖黑褐。
- 头部卡片从“发亮玻璃块”改为“暖色信息牌”。
- 分类按钮、选中态和 hover 态回到统一暖色系。

### 4. 公共控件视觉

联动检查并必要时微调：

- [Button.tsx](/Users/Zhuanz/lvsoso/tools/src/components/common/Button.tsx)
- [TextArea.tsx](/Users/Zhuanz/lvsoso/tools/src/components/common/TextArea.tsx)
- 通用输入框 class

目标是：

- 按钮不再显得像默认 UI 库皮肤。
- 输入框和面板的边界关系一致。
- 焦点态可见，但不过亮。

## 数据与状态影响

- 不修改主题持久化键。
- 不修改 `themeStore` 的行为。
- `default` 主题名称保留，用户现有持久化数据无需迁移。

## 风险与控制

### 风险 1：暖色过重，显得复古过头

控制方式：

- 主背景保持克制的米白和暖灰，不引入过深的黄褐色大面积铺底。
- 强调色只承担操作识别，不承担整页装饰。

### 风险 2：深色模式变脏或对比不够

控制方式：

- 深色模式仍以高可读文本为第一优先。
- 保持按钮、输入框、选中态和正文之间的对比度。

### 风险 3：模块内现有 `bg-card` / `bg-input` 使用点联动过大

控制方式：

- 优先通过令牌收敛风格，而不是逐模块重写 class。
- 修改后以构建和页面抽查确认常用模块没有出现不可读或层级塌陷。

## 验证

最低验证：

- `npm run build`

人工检查重点：

- 首页壳层和侧栏是否已经脱离“偏紫玻璃风”。
- JSON Formatter、Mermaid Viewer、Translator 三个模块是否仍可读、层级清晰。
- 浅色 / 深色模式切换后是否都保持“米白书页 + 书房工作台”的同一主题家族。

## 结论

本次按“主题令牌重做 + 公共样式联动”的中等范围实施，重点修复默认主题的色彩气质、材质语言和公共组件统一性，不扩展到整站结构改版。
