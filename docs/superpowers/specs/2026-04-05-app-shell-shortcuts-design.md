# 应用壳层快捷键设计文档

**日期**: 2026-04-05  
**状态**: 设计已确认，待实现

## 1. 背景

当前工具站已经有左侧菜单、搜索框和主内容区，但还没有统一的全局快捷键能力：

- 左侧菜单只能用鼠标展开和收起
- 搜索框只能手动点击聚焦
- 没有快捷键帮助入口，用户无法发现已有或新增的键位
- 侧边栏如果后续支持收起，也缺少持久化和再发现机制

现有实现集中在以下文件：

- [App.tsx](/Users/Zhuanz/lvsoso/tools/src/App.tsx)
- [Sidebar.tsx](/Users/Zhuanz/lvsoso/tools/src/components/layout/Sidebar.tsx)
- [toolStore.ts](/Users/Zhuanz/lvsoso/tools/src/stores/toolStore.ts)

项目当前没有全局热键封装、弹层组件或本地存储封装，因此本次设计需要在现有结构上做最小扩展。

## 2. 目标

### 2.1 功能目标

- 为应用壳层增加一组轻量、通用、低冲突的全局快捷键
- 支持用 `Cmd/Ctrl + \` 切换左侧菜单显示与隐藏
- 支持用 `/` 快速聚焦左侧搜索框
- 支持用 `?` 打开快捷键帮助面板
- 支持用 `Esc` 关闭快捷键帮助面板
- 用 `localStorage` 持久化左侧菜单展开/隐藏状态

### 2.2 非目标

- 不为各个具体工具页面补快捷键
- 不引入 Vim 风格导航或多段序列键
- 不引入第三方热键库
- 不抽象通用 Modal / Dialog 组件
- 不使用 IndexedDB

## 3. 方案对比

### 方案 A：App 级全局快捷键 + Zustand 壳层状态 + 原生帮助面板

这是最终采用的方案。

优点：

- 与当前项目结构最贴合，改动集中
- 不引入新依赖，维护成本低
- 状态边界清楚，足够覆盖当前需求

缺点：

- 需要在 `App` 层自己处理焦点和键盘事件
- 帮助面板先是页面内专用实现，复用性有限

### 方案 B：额外抽 `useGlobalShortcuts` / `useSidebarPreference`

优点：

- 逻辑更清晰，后续扩展更容易

缺点：

- 对当前体量来说有点提前抽象
- 会把简单壳层需求拆成更多文件

### 方案 C：引入热键库

优点：

- API 完整，支持复杂快捷键管理

缺点：

- 当前需求太小，收益不足
- 增加依赖和使用约束

## 4. 最终设计

### 4.1 快捷键映射

本次只覆盖全局壳层操作：

- `Cmd/Ctrl + \`：切换左侧菜单显示/隐藏
- `/`：聚焦左侧搜索框
- `?`：打开或关闭快捷键帮助面板
- `Esc`：关闭快捷键帮助面板

设计取舍：

- 不使用 `Cmd/Ctrl + B`，避免与文本编辑区的加粗快捷键冲突
- 不使用单字母全局导航，避免误触
- `?` 和 `/` 只在非输入态生效

### 4.2 输入态冲突规避

以下元素聚焦时，不响应 `/` 和 `?`：

- `input`
- `textarea`
- `select`
- `contenteditable`

这样可以避免：

- 用户在搜索框或文本编辑区输入 `/` 时被打断
- 用户在富文本或 Mermaid 编辑区输入 `?` 时意外弹出帮助

`Cmd/Ctrl + \` 作为壳层开关，仍允许全局生效。

### 4.3 侧边栏显示策略

左侧菜单采用“完全隐藏 / 完全展开”，不做窄图标栏。

展开态：

- 保持当前侧边栏结构和宽度
- 显示搜索框、分类和工具列表

隐藏态：

- 整个侧边栏从布局中移除
- 主内容区占满剩余宽度

为了避免隐藏后只能靠记忆快捷键恢复，本次增加再发现入口：

- 当侧边栏隐藏时，在主内容区左上角显示一个轻量浮动按钮
- 按钮文案为“展开菜单”或等价短文案
- 按钮 tooltip 或辅助文案显示 `Cmd/Ctrl + \`

### 4.4 搜索聚焦行为

`/` 聚焦搜索框时遵循以下规则：

- 如果侧边栏当前已展开，直接聚焦搜索框并选中现有内容
- 如果侧边栏当前隐藏，先展开侧边栏，再聚焦搜索框
- 聚焦动作通过 `requestAnimationFrame` 或等价时序保证在输入框可见后触发

这样可以保证 `/` 永远是“去搜索”，而不是因为侧边栏隐藏而失效。

### 4.5 状态管理

继续复用现有 `toolStore`，新增壳层状态：

- `isSidebarOpen`
- `isShortcutHelpOpen`

以及对应操作：

- `setSidebarOpen`
- `toggleSidebar`
- `setShortcutHelpOpen`
- `toggleShortcutHelp`

本次不拆新的 store。

### 4.6 本地持久化

左侧菜单状态持久化到 `localStorage`。

建议 key：

- `devtools.sidebar.open`

行为规则：

- 首次访问时默认展开
- 用户切换后立即写入 `localStorage`
- 应用初始化时读取并恢复

帮助面板不持久化：

- 每次刷新默认关闭
- 不记录“上次是否打开”

### 4.7 快捷键帮助面板

帮助面板采用页面内简单覆盖层，不抽象成通用组件。

内容只展示本次支持的 3 组主快捷键：

- `Cmd/Ctrl + \`：切换侧边栏
- `/`：聚焦搜索
- `?`：打开快捷键帮助

交互要求：

- 支持点击遮罩关闭
- 支持右上角关闭按钮
- 支持 `Esc` 关闭
- 面板本身不需要复杂分组或搜索

### 4.8 视觉与提示

侧边栏展开态建议在标题区或搜索框附近增加轻提示入口，至少满足以下之一：

- 一个小型“快捷键”按钮
- 一个弱提示文案，如 `? 查看快捷键`

目标不是堆信息，而是让用户知道这套功能存在。

## 5. 组件职责边界

- `App`
  - 负责全局快捷键注册
  - 负责主布局根据侧边栏状态切换
  - 负责在侧边栏隐藏时渲染再发现入口
- `Sidebar`
  - 负责现有菜单结构
  - 负责暴露搜索框聚焦能力
  - 负责展示轻量快捷键提示
- `toolStore`
  - 负责壳层可持久化状态

不做的事情：

- 不把工具级快捷键和壳层快捷键混在一个注册器里
- 不顺手重构分类展开逻辑

## 6. 风险与控制

主要风险：

- 全局快捷键误伤输入区
- 侧边栏隐藏后用户找不到恢复入口
- 初始化读取本地状态时出现首屏布局闪动

控制策略：

- 对 `/` 和 `?` 做严格输入态过滤
- 在隐藏态始终保留一个浮动展开按钮
- store 初始化时同步读取 `localStorage`，避免先展开再收起的闪动

## 7. 验证策略

本次以构建和手工回归为主：

- `Cmd/Ctrl + \` 能切换左侧菜单，并在刷新后保持状态
- `/` 能聚焦搜索框；侧边栏隐藏时会先展开再聚焦
- `?` 能打开帮助面板
- `Esc`、遮罩点击、关闭按钮都能关闭帮助面板
- 在搜索框、文本框、Mermaid 编辑区输入 `/`、`?` 时不会被全局快捷键劫持
